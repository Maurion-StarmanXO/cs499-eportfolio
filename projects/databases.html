Databases

Artifact: CS-340 Project Two Animal Shelter Dashboard

This artifact comes from my CS-340 Project Two assignment. The project used MongoDB as the database, Python for the CRUD module, and Dash/Plotly for the dashboard interface. I selected this artifact for the database category because it directly shows database access, data retrieval, filtering, and visualization in a working application.

Enhancement Overview

For this enhancement, I improved the database side of the project by expanding the dataset and making the dashboard more realistic. Before I enhanced this project, the graph was distorted or simply not optimized. I wanted to fix that as part of enhancing the database artifact while also showing the value of adding more data for a more realistic real-world use case.

I added an expanded animal intake dataset that included more animal types such as exotic animals, fish, birds, cats, and dogs. This made the dashboard better represent a broader shelter dataset instead of only focusing on the original limited records. I also improved the visual presentation of the animal type graph so the data was easier to understand.

Skills Demonstrated

This artifact demonstrates MongoDB database usage, CRUD operations, CSV data import, database querying, dashboard integration, and data visualization. It also shows that I can work through setup issues, connect an application to a database, and verify that records are being stored and retrieved correctly.

Course Outcomes Supported

This enhancement supports the course outcomes related to implementing computer solutions, communicating technical information visually, and applying database concepts to a practical problem. It also supports a security mindset because database applications require careful thinking about access, data handling, and system reliability.

This helped me better understand how database-backed applications work in practice.
