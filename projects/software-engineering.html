Artifact: CS-255 System Analysis and Design

This artifact comes from my CS-255 System Analysis and Design course. The project focused on analyzing a client’s needs and translating those needs into system requirements, use cases, and design documentation. I selected this artifact because it shows my ability to think beyond code and approach software development from a planning, design, and user-centered perspective.

Enhancement Overview

For this enhancement, I focused on improving the clarity, organization, and professional presentation of the system design work. The goal was to make the artifact easier to understand for a technical or nontechnical audience. I strengthened the way requirements were communicated, clarified the system purpose, and improved how the design decisions connected back to the user’s needs.

Skills Demonstrated

This artifact demonstrates software design, requirements analysis, technical communication, and system planning. It shows that I can evaluate a problem, identify user needs, and organize those needs into a structured system design. It also reflects my ability to communicate software solutions clearly before development begins.

Course Outcomes Supported

This enhancement supports the course outcomes related to professional communication, designing computing solutions, and using well-founded computer science practices. It also helped me improve how I explain technical decisions in a way that is understandable and useful to stakeholders.

Reflection

Working on this artifact helped me understand that good software starts before code is written. A project can fail if requirements are unclear or if the design does not match what users actually need. This enhancement helped me grow in planning, documentation, and professional communication, which are important skills for my long-term goals in systems administration, DevOps, infrastructure, and security.
